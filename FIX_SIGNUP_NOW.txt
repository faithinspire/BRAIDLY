================================================================================
                    SIGNUP FIX - ACTION REQUIRED
================================================================================

PROBLEM: Signup is not creating accounts

ROOT CAUSE: Supabase RLS policies are missing INSERT permissions

SOLUTION: Add 3 RLS policies to your Supabase database

================================================================================
                         STEP 1: GO TO SUPABASE
================================================================================

1. Open your Supabase project: https://app.supabase.com
2. Go to SQL Editor
3. Create a new query
4. Copy and paste the SQL below
5. Click "Run"

================================================================================
                         STEP 2: RUN THIS SQL
================================================================================

CREATE POLICY "Users can create own profile" ON profiles
  FOR INSERT WITH CHECK (auth.uid() = id);

CREATE POLICY "Braiders can create own record" ON braiders
  FOR INSERT WITH CHECK (auth.uid() = id);

CREATE POLICY "Customers can create own record" ON customers
  FOR INSERT WITH CHECK (auth.uid() = id);

================================================================================
                         STEP 3: VERIFY
================================================================================

After running the SQL:
1. Go to Authentication → Policies
2. You should see 3 new INSERT policies
3. If you see them, the fix worked!

================================================================================
                         STEP 4: TEST SIGNUP
================================================================================

1. Clear browser cache (Ctrl+Shift+Delete)
2. Go to http://localhost:5173/signup
3. Fill in the form
4. Click "Create Account"
5. Should now work!
