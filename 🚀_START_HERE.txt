================================================================================
ROOT CAUSE ANALYSIS
================================================================================

Why was auth broken?

1. RLS policies blocking all access
   - RLS enabled but no policies allowed authenticated users
   - Result: 403 Forbidden on every profile query

2. Profile not auto-created on signup
   - No database trigger to auto-create profile
   - App tried to create but RLS blocked it
   - Result: Orphan auth users

3. Retry loops masking real errors
   - Auth service had 3-attempt retry loops
   - Real errors hidden behind retries
   - Result: Impossible to debug

4. Codebase pollution
   - Multiple conflicting files and schemas
   - Duplicate auth logic
   - Legacy files not cleaned up
   - Result: Confusion and inconsistency

How does the fix work?

1. Clean schema with correct RLS
   - New profiles table with correct structure
   - Database trigger auto-creates profile
   - RLS policies allow authenticated users
   - Result: No 403 errors

2. Strict auth flow
   - No retry loops
   - Explicit error handling
   - One attempt per operation
   - Result: Clear debugging

3. Clean codebase
   - All legacy files deleted
   - No conflicting code
   - Single source of truth
   - Result: Consistency
