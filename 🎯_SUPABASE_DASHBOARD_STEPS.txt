═══════════════════════════════════════════════════════════════════════════════════
STEP 1: OPEN SUPABASE DASHBOARD
═══════════════════════════════════════════════════════════════════════════════════

1. Open browser
2. Go to: https://app.supabase.com
3. Login with your email and password
4. You should see your projects list

═══════════════════════════════════════════════════════════════════════════════════
STEP 2: SELECT YOUR PROJECT
═══════════════════════════════════════════════════════════════════════════════════

1. Look for project: "braidly"
2. Click on it
3. You should see the project dashboard

═══════════════════════════════════════════════════════════════════════════════════
STEP 3: NAVIGATE TO AUTHENTICATION SETTINGS
═══════════════════════════════════════════════════════════════════════════════════

1. Look at the LEFT SIDEBAR
2. Find: "Authentication" (should be near the top)
3. Click on "Authentication"
4. You should see a menu with options

═══════════════════════════════════════════════════════════════════════════════════
STEP 4: OPEN EMAIL PROVIDER SETTINGS
═══════════════════════════════════════════════════════════════════════════════════

1. In the Authentication menu, look for: "Providers"
2. Click on "Providers"
3. You should see a list of providers (Email, Google, GitHub, etc.)
4. Find: "Email" in the list
5. Click on "Email"

═══════════════════════════════════════════════════════════════════════════════════
STEP 5: DISABLE EMAIL CONFIRMATION
═══════════════════════════════════════════════════════════════════════════════════

1. You should now see the Email provider settings
2. Look for a checkbox labeled: "Confirm email"
3. This checkbox should be CHECKED (enabled)
4. ⭐ CLICK ON IT TO UNCHECK IT
5. The checkbox should now be UNCHECKED (disabled)

═══════════════════════════════════════════════════════════════════════════════════
STEP 6: SAVE THE CHANGES
═══════════════════════════════════════════════════════════════════════════════════

1. Look for a "Save" button (usually at the bottom or top right)
2. Click "Save"
3. You should see a success message
4. The setting is now saved

✅ EMAIL CONFIRMATION IS NOW DISABLED

═══════════════════════════════════════════════════════════════════════════════════
STEP 7: VERIFY THE CHANGE
═══════════════════════════════════════════════════════════════════════════════════

1. Refresh the page (F5)
2. Go back to Authentication → Providers → Email
3. Verify: "Confirm email" checkbox is UNCHECKED
4. ✅ Confirmed!

═══════════════════════════════════════════════════════════════════════════════════
WHAT THIS DOES
═══════════════════════════════════════════════════════════════════════════════════

BEFORE (Email confirmation ENABLED):
- User signs up
- Supabase sends verification email
- User must click link in email to verify
- Only then can user login
- ❌ "email not confirmed" error if they try to login before verifying

AFTER (Email confirmation DISABLED):
- User signs up
- User can login immediately
- No email verification required
- ✅ Seamless signup/login

TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════════

If you can't find the "Confirm email" checkbox:
1. Make sure you're in: Authentication → Providers → Email
2. Scroll down if needed
3. Look for any checkbox related to email confirmation
4. It might be labeled differently (e.g., "Require email verification")

If the setting doesn't save:
1. Check your internet connection
2. Try again
3. Refresh the page
4. Check if you have permission to edit settings

If you still see "email not confirmed" error after disabling:
1. Clear browser cache (Ctrl+Shift+Delete)
2. Try in a new incognito window
3. Make sure schema was deployed
4. Check Supabase logs for errors
