CRITICAL FIXES APPLIED:
========================

1. AuthContext.jsx - Fixed initialization state
   - Changed: initialized = true (WRONG - caused race conditions)
   - To: initialized = false (CORRECT - waits for localStorage check)
   - Added: setInitialized(true) in finally block
   - Result: ProtectedRoute now waits for auth to fully initialize

2. Login.jsx - Fixed role-based redirect
   - Changed: Always redirected to /customer/dashboard
   - To: Reads user role from localStorage and redirects accordingly
   - Supports: customer, braider, admin roles
   - Result: Users now go to correct dashboard after login

WHY THIS FIXES THE LOADING LOOP:
================================
- Before: ProtectedRoute checked initialized=true immediately
- But: User data wasn't loaded from localStorage yet
- Result: Infinite loading state
- Now: ProtectedRoute waits for localStorage to load first
- Then: Redirects to correct dashboard based on role
