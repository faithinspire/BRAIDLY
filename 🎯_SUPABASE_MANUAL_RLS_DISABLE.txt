🔥 MANUAL RLS DISABLE - STEP BY STEP WITH SCREENSHOTS

THE PROBLEM:
You're getting "permission denied for schema public" - RLS is STILL ENABLED

THE SOLUTION:
Disable RLS manually in Supabase Dashboard (not SQL Editor)

STEPS:

1. Go to: https://app.supabase.com
2. Login with your account
3. Click on "braidly" project
4. Left sidebar → Click "Authentication"
5. Click "Policies" tab
6. You should see RLS policies listed

ALTERNATIVE METHOD (Easier):

1. Go to: https://app.supabase.com
2. Select "braidly" project
3. Left sidebar → Click "SQL Editor"
4. Click "New Query"
5. Copy EXACTLY this (one line at a time if needed):

ALTER TABLE public.profiles DISABLE ROW LEVEL SECURITY;

6. Click "Run"
7. Wait for success
8. Repeat for each table:

ALTER TABLE public.braider_profiles DISABLE ROW LEVEL SECURITY;
ALTER TABLE public.services DISABLE ROW LEVEL SECURITY;
ALTER TABLE public.portfolio_images DISABLE ROW LEVEL SECURITY;
ALTER TABLE public.gallery_images DISABLE ROW LEVEL SECURITY;
ALTER TABLE public.bookings DISABLE ROW LEVEL SECURITY;
ALTER TABLE public.payments DISABLE ROW LEVEL SECURITY;
ALTER TABLE public.reviews DISABLE ROW LEVEL SECURITY;
ALTER TABLE public.verification_documents DISABLE ROW LEVEL SECURITY;
ALTER TABLE public.notifications DISABLE ROW LEVEL SECURITY;

VERIFY IT WORKED:

1. New Query
2. Paste:

SELECT tablename FROM pg_tables WHERE schemaname='public' AND rowsecurity=true;

3. Click Run
4. If result is EMPTY = SUCCESS ✅
5. If result shows tables = RLS still enabled ❌

AFTER SUCCESS:
- Hard refresh: Ctrl+Shift+R
- Clear cache: Ctrl+Shift+Delete
- Test login/signup
